## 一、开发环境要求

| 组件 | 版本 | 用途 |
|------|------|------|
| JDK | 21+ (Microsoft Build) | 后端运行时 |
| Maven | 3.9+ | 项目构建 |
| Node.js | 20 LTS | 前端构建 |
| MySQL | 8.0+ | 关系型数据库 |

| IDE | IntelliJ IDEA 2025+ | 推荐开发工具 |

## 二、项目模块结构

```
socialflow/                          # 父 POM
├── socialflow-common/               # 公共模块：工具类、常量、枚举、异常
├── socialflow-model/                # 数据模型：Entity、DTO、VO
├── socialflow-dao/                  # 数据访问：MyBatis-Plus Mapper
├── socialflow-service/              # 业务逻辑：AI、内容、知识库、发布等
├── socialflow-web/                  # Web 层：Controller、拦截器、异常处理
├── socialflow-admin/                # 启动模块：Spring Boot Application
├── socialflow-ui/                   # 前端：Vue 3 + TypeScript + Element Plus
├── docs/                            # 项目文档
└── sql/                             # 数据库脚本
```

### 模块依赖关系（单向，从上到下）

```
admin → web → service → dao → model → common
```

## 三、命名规范

### 3.1 数据库

| 类别 | 规范 | 示例 |
|------|------|------|
| 表名 | 小写下划线 | `sys_user`, `content`, `media_asset` |
| 字段名 | 小写下划线 | `user_id`, `create_time`, `is_deleted` |
| 主键 | Snowflake 雪花 ID | `BIGINT NOT NULL` |
| 字符集 | utf8mb4_general_ci | 支持 emoji |

### 3.2 Java 代码

| 类别 | 规范 | 示例 |
|------|------|------|
| Entity | PascalCase | `Content`, `MediaAsset`, `EvalTask` |
| DTO | XxxDTO | `ContentGenerateDTO`, `LoginDTO` |
| VO | XxxVO | `ContentVO`, `EvalReportVO`, `ImagePromptVO` |
| Controller | XxxController | `ContentController`, `ImageController` |
| Service 接口 | XxxService | `ContentService`, `ImageService` |
| Service 实现 | XxxServiceImpl | `ContentServiceImpl`, `ImageServiceImpl` |
| Mapper | XxxMapper | `ContentMapper`, `MediaAssetMapper` |

### 3.3 REST API

| 规范 | 说明 |
|------|------|
| 路径前缀 | `/api/v1/{module}` |
| HTTP 方法 | GET(查询) POST(创建/操作) PUT(更新) DELETE(删除) |
| 认证 | `Authorization: Bearer {token}`（Sa-Token） |
| 响应格式 | 统一 `R<T>` 包装 |

## 四、统一响应体

```java
public class R<T> {
    private int code;           // 状态码（200=成功，其他=失败）
    private String message;     // 提示信息
    private T data;             // 数据体
    private long timestamp;     // 时间戳
    private boolean success;    // 是否成功
}
```

## 五、全局异常处理

| 异常类 | HTTP 状态码 | 说明 |
|--------|------------|------|
| `NotLoginException` | 401 | Sa-Token 未登录 |
| `NotFoundException` | 404 | 资源不存在 |
| `ParamException` | 400 | 参数校验失败 |
| `BusinessException` | 500 (code=1xxx) | 业务逻辑异常 |
| `AiCallException` | 500 (code=2001) | AI 模型调用失败 |
| `GuardrailException` | 500 (code=2002) | 内容安全拦截 |
| `QuotaExceededException` | 500 (code=2003) | 配额超限 |
| `Exception`（兜底） | 500 | 返回类名+消息便于调试 |

## 八、配置项说明

### 8.1 AI 模型配置 (`socialflow.ai`)

```yaml
socialflow:
  ai:
    default-provider: deepseek          # 默认 LLM 提供商
    system-api-key: sk-xxx              # DeepSeek API Key
    providers:
      deepseek:
        base-url: https://api.deepseek.com/v1
        default-model: deepseek-reasoner
        timeout: 300s
      qwen:
        base-url: https://dashscope.aliyuncs.com/compatible-mode/v1
        default-model: qwen-plus
        timeout: 120s
```

### 8.2 AI 文生图配置 (`socialflow.image`)

```yaml
socialflow:
  image:
    dashscope-api-key: sk-xxx           # DashScope API Key（与 embedding 共用）
    dashscope-base-url: https://dashscope.aliyuncs.com/api/v1
    model: wanx-v1                      # 通义万相模型
    image-count: 4                      # 每次生成图片数
    image-size: 1024*1024               # 图片尺寸
    poll-interval-ms: 3000              # 轮询间隔
    max-poll-count: 40                  # 最大轮询次数
```

### 8.3 向量与 Embedding 配置

```yaml
socialflow:
  embedding:
    model: text-embedding-v3            # Qwen 文本嵌入模型
    api-key: sk-xxx                     # DashScope API Key
    dimension: 1024                     # 向量维度
  vector:
    type: pgvector
    host: localhost
    port: 5432
    database: socialflow_vec
```

### 8.4 RAG 配置 (`socialflow.rag`)

```yaml
socialflow:
  rag:
    chunk-size: 512                     # 分块大小
    chunk-overlap: 64                   # 分块重叠
    top-k: 10                           # 检索数量
    enable-hyde: true                   # HyDE 查询改写
    enable-hybrid-search: true          # 混合搜索（向量+BM25）
    enable-rerank: false                # 重排序
```

### 8.5 存储配置 (`socialflow.storage`)

```yaml
socialflow:
  storage:
    type: minio
    endpoint: http://localhost:9000
    access-key: minioadmin
    secret-key: minioadmin123
    bucket: socialflow
```

## 十、构建与运行

### 后端

```bash
# 编译
mvn compile

# 安装到本地仓库
mvn install -DskipTests

# 启动
mvn spring-boot:run -pl socialflow-admin
```

### 前端

```bash
cd socialflow-ui

# 安装依赖
npm install

# 开发模式
npm run dev

# 生产构建
npm run build

# 类型检查
npx vue-tsc --noEmit
```
