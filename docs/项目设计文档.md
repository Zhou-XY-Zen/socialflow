# SocialFlow 项目设计文档 V2.0

> 更新日期：2026-04-14
> 项目定位：AI 社交媒体内容运营平台

---

## 一、项目概述

SocialFlow 是一个基于 AI 的社交媒体内容创作与运营平台，覆盖从内容创意、文案生成、配图制作到多平台分发的完整工作流。平台深度集成多种 AI 工程能力，旨在提升社交媒体运营效率。

### 1.1 核心目标

- 一站式管理小红书、抖音、朋友圈、公众号等主流平台的内容创作
- AI 驱动的文案生成（支持 RAG 知识增强、Multi-Agent 协作、提示词模板）
- AI 文生图自动配图（DashScope 通义万相）
- 内容质量 A/B 评测（LLM-as-Judge）
- 多平台一键分发（辅助/自动发布）

### 1.2 AI 能力矩阵

| AI 能力 | 实现方式 | 状态 |
|---------|---------|------|
| LLM 文案生成 | DeepSeek-reasoner/chat + Qwen + GLM | ✅ 已实现 |
| RAG 知识增强 | HyDE + pgvector + BM25 + RRF 融合 | ✅ 已实现 |
| Multi-Agent 协作 | 策划→写作→审核→优化 四阶段流水线 | ✅ 已实现 |
| AI 文生图 | DashScope wanx-v1 异步文生图 | ✅ 已实现 |
| 内容安全 | 敏感词 + LLM 审核责任链 | ✅ 已实现 |
| A/B 评测 | LLM-as-Judge 多维度评分 | ✅ 已实现 |
| 流式输出 | SSE (Server-Sent Events) | ✅ 已实现 |
| 向量搜索 | PostgreSQL pgvector (1024维) | ✅ 已实现 |
| 提示词工程 | 模板系统 + 变量渲染 + Few-Shot | ✅ 已实现 |

## 三、业务功能设计

### 3.1 内容生成

**支持模式：**
- 单平台生成（同步/流式）
- 多平台批量生成（并行）
- RAG 知识增强生成
- Multi-Agent 协作生成
- AI 改写

**支持平台：**
- 小红书（种草/测评/探店/教程/旅行攻略）
- 抖音（视频描述/话题标签）
- 朋友圈（简洁正式风格）
- 公众号（长文章/图文格式）

**生成管道（5 步）：**
1. 输入安全检查（Guardrail）
2. RAG 知识检索（可选）
3. 提示词模板渲染
4. LLM 调用（DeepSeek/Qwen/GLM）
5. 输出安全检查 + 持久化 + 向量化

### 3.2 AI 智能配图

**技术方案：** DashScope 通义万相 wanx-v1 文生图

**流程：**
1. 文案生成完成后，DeepSeek-chat 分析文案内容生成中文绘图提示词
2. 用户可编辑提示词
3. 调用 DashScope 异步 API 生成 4 张图片
4. 前端每 3 秒轮询任务状态
5. 生成完成后展示图片，用户勾选
6. 下载选中图片到 MinIO，创建 MediaAsset 记录
7. 自动生成标签（主题+平台+场景词）
8. 绑定到文案（content_media_rel）

### 3.3 知识库与 RAG

**文档处理流程：**
1. 上传文档（PDF/Word/TXT）→ MinIO 存储
2. 异步解析文本内容
3. 按 512 字符分块（64 字符重叠）
4. 每块调用 Qwen text-embedding-v3 向量化
5. 存入 pgvector 集合

**RAG 检索管道（8 步）：**
1. HyDE 查询改写（用 LLM 生成假设文档）
2. 查询文本向量化
3. pgvector 向量相似搜索
4. MySQL BM25 全文搜索
5. RRF (Reciprocal Rank Fusion) 结果融合
6. 可选：BGE-Reranker 重排序
7. 元数据富化
8. 上下文注入到提示词

### 3.4 Multi-Agent 协作

**四阶段流水线：**

| 阶段 | Agent | 职责 | 输出 |
|------|-------|------|------|
| 1 | Planner（策划） | 分析主题，制定内容策略 | JSON 计划 |
| 2 | Writer（写手） | 根据计划撰写初稿 | 文案正文 |
| 3 | Reviewer（审核） | 审核质量，指出问题 | JSON 评审意见 |
| 4 | Optimizer（优化） | 根据意见优化文案 | 优化后正文 |

- Reviewer 可以驳回 → Optimizer 迭代（最多 maxRounds 轮）
- 仅 Writer 和 Optimizer 的输出推送到前端
- Planner/Reviewer 的输出作为阶段事件显示

### 3.5 提示词模板系统

**模板结构：**
- system_prompt：AI 角色定义
- user_prompt_template：用户提示词（含 `{{变量}}` 占位符）
- few_shot_examples：少样本示例（JSON 数组）
- variables：变量定义（JSON Schema）

**支持变量：**
`{{topic}}`, `{{keywords}}`, `{{wordCount}}`, `{{platform}}`, `{{tone}}`, `{{productInfo}}`

**字数控制机制：**
- 兜底模板包含"【字数硬限制】全文必须控制在 {{wordCount}} 字以内"
- 数据库模板全部使用动态 `{{wordCount}}`
- System Prompt 强调"必须严格遵守字数限制"

### 3.6 A/B 评测系统

**评测流程：**
1. 创建评测任务（配置 A vs 配置 B + 测试主题列表）
2. 异步执行：逐个主题分别用 A/B 配置生成文案
3. LLM-as-Judge 对每对文案打分（6 个维度）
4. 汇总报告：胜负统计 + 维度对比 + 最佳/最差案例

**评分维度（加权）：**
- 内容相关性 relevance (0.25)
- 平台风格 style (0.25)
- 语言流畅 fluency (0.15)
- 创意性 creativity (0.15)
- 格式规范 format (0.10)
- 忠实度 faithfulness (0.10)

**偏差消除：** 随机交换 A/B 顺序，消除位置偏差

### 3.7 多平台分发

**发布模式：**
- **辅助发布**（所有平台）：格式化文案 + 一键复制 + 下载配图
- **自动发布**（已认证公众号）：草稿+发布 API

**微信公众号三级降级：**
1. 草稿+发布（`draft/add` + `freepublish`）—— 需认证
2. 图文群发（`uploadnews` + `mass/sendall`）—— 需认证
3. 纯文本群发（`mass/sendall` text）—— 基础权限

**内容格式处理：**
- MarkdownUtil.stripMarkdown()：去除标记 → 纯文本
- MarkdownUtil.markdownToHtml()：转换 → HTML 富文本
- 自动生成封面图（900×383 渐变蓝色 + SocialFlow 标识）

### 3.8 内容安全

**GuardrailService 责任链：**
1. 输入长度校验
2. 敏感词过滤（字典匹配）
3. LLM 输出审核
4. 平台合规检查
5. 审核日志记录

## 四、数据库设计

### 4.1 核心表关系

```
sys_user (1) ──── (N) content
sys_user (1) ──── (N) media_asset
sys_user (1) ──── (N) knowledge_base
content  (M) ──── (N) media_asset    [via content_media_rel]
content  (1) ──── (N) content_version
knowledge_base (1) ─── (N) kb_document
kb_document (1) ─── (N) kb_chunk
eval_task (1) ──── (N) eval_result
```

### 4.2 设计原则

- **雪花 ID**：MyBatis-Plus 自动生成，JacksonConfig Long→String 避免 JS 精度丢失
- **逻辑删除**：`is_deleted` 字段，MyBatis-Plus 自动过滤
- **自动填充**：`create_time`, `update_time` 由 MetaObjectHandler 填充
- **JSON 存储**：`generation_params`, `configA`, `configB`, `test_topics` 等灵活字段用 JSON 类型

## 五、安全设计

### 5.1 认证

- Sa-Token JWT Token，`Bearer` 前缀
- Token 有效期 7 天
- Redis 存储会话状态

### 5.2 数据安全

- API Key 使用 AES-256-GCM 对称加密存储
- 密码使用 BCrypt 哈希
- MinIO 桶策略：仅 `media/*` 路径公开读取

### 5.3 内容安全

- 输入/输出双向 Guardrail 检查
- 敏感词字典匹配
- LLM 辅助审核
- 完整审核日志
